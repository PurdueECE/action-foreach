git tag 1.0 <commit id> used to capture and tag a specific point in history using commit id
git reset <filename> used to undo local changes of the file to the Git repository
git stash list shows a list of the repository's stashes which are where the temporary changes made on a working copy are stored
git stash pop takes the most recent change on the repository and removes it from the stack and adds onto the top of the current working tree
git rm -f <filename> removes a file even when file is not up to date with the most recent commit because of the -f flag
