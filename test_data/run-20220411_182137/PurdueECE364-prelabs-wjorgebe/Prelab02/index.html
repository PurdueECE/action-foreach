git tag 1.0 <commit id> Git tags are used to keep track of certain commits in the git history, commonly used to mark release points.
git reset <filename> Git reset can be used to undo local changes made to a file locally to get back to a a version from a previous commit.
git stash list The git stash list keeps track of all the stashes of local changes you have made that you can later use and reimplement.
git stash pop Stash pop takes a stashed state from the stash list (see above) and applies it to the current working tree.
git rm -f <filename> This command basically removes a specified file everywhere. This means the file is deleted form the repository as well as from the current local file system.
