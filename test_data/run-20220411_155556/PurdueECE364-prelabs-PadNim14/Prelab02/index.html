command purpose
git tag 1.0 <commit id> used when needing to have specific reference points in the git history
git reset <filename> takes the current file and removes it from the staging area
git stash list temporarily stores the changes you made to file(s) and allowing the user to work on something else
git stash pop pulls the most recent stash from the git history, makes the respective changes, and deletes the stash entry from the git history
git rm -f <filename> deletes the file from the git repository
